# GPXroute: 校园跑路径.gpx生成及预览
> 一个基于数学建模的地理位置仿真工具，专门用于生成标准 400 米 / 200 米田径场的 GPX 运动轨迹。支持任意角度旋转、坐标微调平移以及 GPS 信号抖动模拟。
---

## 核心功能
几何建模：严格遵循标准田径场（80m 直道 + 36m 半径弯道）进行点位计算，非简单的圆形绕圈。

线性空间变换：

- 旋转矩阵：支持 heading_deg 参数，通过 2D 旋转矩阵算法将标准坐标系映射至真实操场方位。

- 起点平移：提供 offset_x 与 offset_y 修正，解决地图坐标偏移（如 GCJ-02 偏差）导致的“撞墙”问题。

生物特征仿真：

- 动态配速：根据目标长度与速度自动分配每一秒的位移。

- 高斯噪声：内置 ±0.2m 的随机漂移，模拟真实运动中的 GPS 信号抖动。

可视化验证：自动生成基于 Folium 的交互式 HTML 地图，生成后即可预览轨迹是否对准跑道。

## 环境依赖
根目录的 requirements.txt已描述了需要的库，执行以下命令配置环境：
```Bash
pip install -r requirements.txt
```

## 参数说明
运行脚本后，可通过交互式命令行输入以下参数（直接回车使用默认值，默认值为紫金港西操）：

- `start_lat`，`start_lon`: 起始纬度，经度（起始点为直跑道起点）
- `target_dist`: 目标跑距（单位：米；建议稍大于实际要求）
- `target_speed`: 目标跑速（单位：米/s；默认数据换算后为6min/km） 
- `heading_deg`: 偏移角度（单位：度；初始指向正西，逆时针偏移）
- `offset_x`，`offset_y`: 偏移坐标（单位：米；正值对应西/北）

## 使用流程
执行脚本并按照提示输入。

位置校准：
观察输出的 Offset Result 坐标，并在 preview.html 中确认蓝线是否完美契合操场。

Xcode 仿真：
将生成的 route.gpx 导入 Xcode，通过 Simulate Location 开始仿真。

## 输出文件
route.gpx：符合标准 GPX 1.1 协议的轨迹文件，包含时间戳信息。

preview.html：交互式地图预览文件，建议在浏览器中打开检查。
